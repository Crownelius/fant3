# MIX V3 NVIDIA ‚Äî 11-Source Reasoning-Heavy Mix

**Used for:** 150m, 350m, 742m, and 1b training scales in `notebooks/fant3_colab_train.ipynb` (cell 12, selected when `TARGET_SCALE` is not `20m` or `50m`).

**Purpose:** Maximise reasoning benchmark performance at larger parameter counts where NVIDIA's large-scale CoT (Chain of Thought) corpora provide value. NVIDIA datasets represent roughly 60% of the mix by weight. FineWeb-Edu (20%) keeps general language statistics healthy.

**Total weight:** 1.00 (weights re-normalised if any source fails to load).

## NVIDIA dataset notes

All NVIDIA datasets in this mix are published under CC-BY-4.0. The practical attribution requirement is to acknowledge the datasets in the model card at release time ‚Äî the licence is otherwise equivalent to permissive open-source for research use.

Three large NVIDIA corpora are **not included** because they are gated (access request required on the HuggingFace dataset page):
- `nvidia/Nemotron-CC-v2.1` ‚Äî the 6.3 T-token web corpus
- `nvidia/Nemotron-CC-Math-v1` ‚Äî 85 B math extraction
- `nvidia/Nemotron-CC-Code-v1` ‚Äî 52 B code extraction

### `OpenMathReasoning` split selection

The `cot` split is selected (5.68 M rows). Other available splits ‚Äî `tir`, `genselect`, `additional_problems` ‚Äî are not used. The `cot` split contains standard chain-of-thought solutions without tool-integrated reasoning, making it directly compatible with FANT 3's `PROBLEM_SOLUTION` format handler.

### `OpenCodeReasoning-2` split selection

The `python` split is selected (2.16 M rows). The `cpp` split is not used.

### `Nemotron-Cascade-2-SFT-Data` config requirement

This dataset requires an explicit config name per subset. Each of the four subsets (`math`, `chat`, `instruction_following`, `science`) is loaded as a separate entry in the mix with its own weight.

## Format notes

- **PROBLEM_SOLUTION** (OpenMathReasoning, OpenMathInstruct-2, OpenCodeReasoning-2): `extract_text()` falls back from `solution` ‚Üí `generated_solution` ‚Üí `answer`. OpenMathInstruct-2 uses `generated_solution`; the fallback logic in `formats.py` handles this transparently.
- **PROBLEM_THINK_SOLUTION** (Crownelius): emits `<|think|>...<|/think|>\n<|answer|>...<|/answer|>` for format alignment.
- **MESSAGES** (Cascade-2, Sonnet): both OpenAI and ShareGPT role formats are normalised.

## NeMo-style training recipe

When using MIX V3, the notebook applies NVIDIA NeMo-inspired hyperparameters:
- Warmup 500 steps (vs the previous 200) ‚Äî safer for MoE (Mixture of Experts) routing stabilisation
- Peak LR (Learning Rate) 1.5 e-4 to 2.0 e-4 depending on scale
- Gradient accumulation 4 ‚Üí effective batch 8 ‚Äî matches the Nemotron intent of ~4 M tokens/step at smaller batch sizes
- AdamW betas (0.9, 0.95), weight_decay 0.1, eps 1e-8 ‚Äî all NVIDIA defaults
