# ADR 0003: NVIDIA Reasoning Datasets as Primary Training Signal

## Status

Accepted (implemented 2026-04-19 as MIX v3)

---

## Context

FANT 3's initial training mix (MIX v1, early 2026-04-19 runs) used community-sourced HF (HuggingFace) datasets:

- FineWeb-Edu (general web text filtered for educational content)
- Crownelius/Opus-4.6-Reasoning (distilled reasoning traces from Claude Opus 4.6)
- Kimi K2.5 General-Distillation (distilled from Kimi K2.5)
- NuminaMath-CoT (competition math with chain-of-thought)
- FineTome-100k (curated instruction following)
- Alibaba Superior Reasoning SFT (stage 1)

After three 150m validation runs (steps 750–2500), MMLU accuracy was at statistical chance (26.5%, CI [20.87%, 33.02%]) and GSM8K accuracy was not yet meaningfully above 0.

The core question: is this because the model is undertrained, the architecture is wrong, or the training signal is weak?

The Campaign N ablation on FANT 2 had established that at 5M training scale, **auxiliary losses and format changes hurt** — the only levers that work are structural (SleepGate memory consolidation). This suggested the bottleneck was training signal quality, not architecture.

Simultaneously, NVIDIA released their full open-weight reasoning stack on HuggingFace:
- nvidia/OpenMathReasoning: 5.68M rows of math chain-of-thought generated by DeepSeek-R1
- nvidia/OpenCodeReasoning-2: 2.16M Python chain-of-thought
- nvidia/Nemotron-Cascade-2-SFT-Data: 15.9M rows across 8 configs (math, chat, science, instruction following, safety, etc.)
- nvidia/OpenMathInstruct-2: 14M math instruction pairs (Nemotron-4-340B teacher)

All datasets are CC-BY-4.0 license.

---

## Decision

Replace the community-focused mix with **MIX v3** giving NVIDIA datasets 60% weight, and apply the **NeMo-style training recipe**:

**Data composition (MIX v3):**

| Source | Weight | Rationale |
|---|---|---|
| nvidia/OpenMathReasoning [cot] | 0.18 | Largest volume; DeepSeek-R1 quality |
| nvidia/OpenCodeReasoning-2 [python] | 0.08 | Code reasoning chain-of-thought |
| nvidia/OpenMathInstruct-2 | 0.08 | Math instruction; Nemotron-4-340B teacher |
| nvidia/Nemotron-Cascade-2 [math] | 0.08 | Curated math SFT (Supervised Fine-Tuning) |
| nvidia/Nemotron-Cascade-2 [chat] | 0.06 | Chat conversation quality |
| nvidia/Nemotron-Cascade-2 [if] | 0.04 | Instruction following |
| nvidia/Nemotron-Cascade-2 [science] | 0.04 | Science domain |
| nvidia/Daring-Anteater | 0.04 | Curated 100k instruction following |
| HuggingFaceFW/fineweb-edu | 0.20 | General text backbone |
| claude-sonnet-4.6-120000x | 0.12 | Sonnet 4.6 distillation |
| Opus-4.6-Reasoning | 0.08 | Opus 4.6 chain-of-thought |

**NeMo-style recipe changes:**

| Knob | Before | After | Reason |
|---|---|---|---|
| WARMUP_STEPS | 200 | 500 | NVIDIA uses 2–5% of total; extra-safe for MoE |
| peak_lr (LR) | 1.5e-4 | 2.0e-4 | Higher but safe (previous NaN was at 3.0e-4) |
| GRAD_ACCUM | 1 | 4 | Effective batch = 8; closer to NVIDIA's intent |

**Decontamination:** 13-gram SHA-1 filter against GSM8K + MATH-500 + MMLU test sets (457,910 unique hashes). Contamination rates on NVIDIA sources: OpenMathReasoning 0.00%, OpenCodeReasoning-2 0.00%, Cascade-2 math 0.67%, Cascade-2 chat 1.67%, Daring-Anteater 0.33%.

**License note:** All NVIDIA HF datasets are CC-BY-4.0 (attribution only). Practically equivalent to Apache 2.0 for research use. The OpenMathInstruct-2 note about Nemotron-4-340B being trained on benchmark-adjacent problems is disclosed in our dataset registry but does not affect our use — we apply the decontamination filter.

**Evidence that MIX v3 works:** The 742m Tier C run (81.92M tokens) showed domain vocabulary acquisition at CE 5.72: tokens `mutex`, `idempotency`, `INTEGER`, `preimage`, `Tensor Conversion` from the NVIDIA corpus. The Q5 quality probe ("how many pizza slices are left?") correctly predicted `INTEGER` as the domain — the model connected a word problem to a math context. This was not observed in the community-mix 150m runs.

---

## Consequences

**Benefits:**
- Large-volume, high-quality reasoning traces at 60% weight
- CC-BY-4.0 is unambiguously permissive for research
- Contamination rates remain manageable (<2%) with the 13-gram filter
- The NVIDIA NeMo recipe provides battle-tested LR / warmup / accumulation defaults
- SpinorApollonian chirality shows `pre-saturation pattern 0.676 → 0.500` confirming the NVIDIA math signal is exercising both memory packs

**Drawbacks:**
- Three NVIDIA datasets are gated (Nemotron-CC-v2.1 6.3T tokens / Math-v1 85B / Code-v1 52B) — require HF access request; currently using FineWeb-Edu as placeholder at 20%
- OpenMathInstruct-2 has 1.80% contamination (highest in mix) — mitigated by filter but not zero
- schema discrepancy: OpenMathReasoning uses `generated_solution` key, not `solution` — required a PROBLEM_SOLUTION format fix in `fant2/data/formats.py`

**For 20m and 50m scales:** MIX v4 (chat-focused) replaces v3 as the default for those scales. At small scales, chat quality is more tractable than math reasoning and provides a better training signal per training token.

---

## Alternatives Considered

**Community mix only (MIX v1)**  
The baseline. MMLU at chance after 750 steps on 150m. Insufficient reasoning signal volume.

**Distillation-only (Sonnet + Opus + Kimi)**  
Considered: high quality per token but limited volume (2,160 Opus traces available, 120K Sonnet). Not enough training data for 150m+ scales.

**Red Pajama / Dolma / The Pile (open pretraining corpora)**  
Not pursued: general web text has low reasoning density. FineWeb-Edu (educational filtered) is already in the mix at 20%.

**Synthetic procedural math (FANT 2 ProceduralMathStream)**  
Used in FANT 2 training (Campaign N). Not added to FANT 3 mix because: (a) it was confirmed to not generalize to GSM8K at FANT 2 scale, and (b) NVIDIA OpenMathReasoning provides a larger, higher-quality alternative. The `Never train on public benchmarks` rule remains in force — procedural generation stays as an option for augmentation.
