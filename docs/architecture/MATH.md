# FANT 3 Mathematical Foundations

This document explains the mathematical ideas behind FANT 3's memory system. The primary reference is Kocik (2001), arxiv:2001.05866, which establishes the connection between Clifford algebra Cl(2,1), spinors, Apollonian circle packings, and the Descartes circle theorem. Readers familiar with linear algebra but not with Clifford algebras should be able to follow the presentation below.

---

## 1. Apollonian Circle Packings

An **Apollonian packing** is a fractal arrangement of circles constructed by the following rule: given three mutually tangent circles, there exist exactly two circles tangent to all three (Apollonius of Perga, ~200 BC). One of these lies inside the curvilinear triangle formed by the three; the other surrounds all three. Repeatedly inserting the inner circle and applying the same rule generates a dense fractal filling of the original region.

Apollonian packings have three properties that make them useful as a cognitive memory model:

1. **Scale-free (fractal) structure**: the circle sizes follow a power law with exponent approximately 1.305 (Boyd 1982; Mallows 2010). This matches empirical observations about cortical column sizes, episodic memory decay, and schema abstraction.

2. **Two complementary sub-packings**: every Apollonian packing decomposes into two interleaved sub-packings that fill each other's gaps. FANT identifies these with **α memory** (high curvature = small circles = specific instances) and **β memory** (low curvature = large circles = general schemas). This is the brain-level hypothesis: episodic memory fills the gaps of semantic memory and vice versa (Tulving 1985; Hassabis et al. 2007).

3. **Group structure**: the Apollonian group is generated by four reflections, each inverting one of the four mutually tangent circles in a Descartes configuration. The group acts on the space of 4-tuples of curvatures by 4×4 integer matrices, preserving the Descartes quadratic form.

## 2. The Descartes Circle Theorem

Given four mutually tangent circles with curvatures (= reciprocal radii) b₁, b₂, b₃, b₄, the Descartes circle theorem states:

```
(b₁ + b₂ + b₃ + b₄)² = 2(b₁² + b₂² + b₃² + b₄²)
```

This is equivalent to saying that the 4-vector **b** = (b₁, b₂, b₃, b₄) lies on the zero-set of the **Minkowski quadratic form** in signature (1, 3):

```
Q(b) = (Σᵢ bᵢ)² − 2 Σᵢ bᵢ² = 0
```

The group SO(3,1) (Lorentz group, or equivalently the Apollonian group) acts on this quadric, preserving Q(**b**) = 0. This is a purely algebraic reformulation: Apollonian packings are orbits of SO(3,1) on the Descartes quadric.

The **Descartes regularizer** in `SpinorApollonianMemory.descartes_loss()` measures how far a local 4-spinor neighborhood is from satisfying this equation. When training nudges memory representations toward the quadric, the memory self-organizes into an Apollonian geometry.

## 3. Clifford Algebra Cl(2,1) and the Minkowski Spinor

A Clifford algebra Cl(p, q) is an associative algebra generated by a vector space V = ℝ^(p+q) with a quadratic form of signature (p, −q), subject to the relation:

```
v² = Q(v) · 1     for all v ∈ V
```

For **Cl(2, 1)** (two positive dimensions, one negative), the generating space is ℝ³ with the Minkowski inner product of signature (1, −1, −1). The spinor representation of Cl(2, 1) is a 2D complex (or real) space ℝ² with the quadratic form:

```
Q_Cl(s) = s₀² − s₁²     (signature (1, −1))
```

Kocik (arxiv:2001.05866, §3) shows that every **tangent pair** of circles in an Apollonian packing corresponds to a Minkowski spinor s = (s₀, s₁) ∈ ℝ² satisfying Q_Cl(s) ≥ 0. The **sign of s₁** determines which of the two complementary sub-packings the circle belongs to:

```
s₁ > 0  →  left-chiral circle  (sub-packing α)
s₁ ≤ 0  →  right-chiral circle (sub-packing β)
```

This chirality split is exactly what `SpinorApollonianMemory` uses to route hidden states into the α and β memory packs. The scalar threshold of the legacy `ApollonianMemory` tried to use a single number (the L2 norm) to accomplish this split; the spinor uses two numbers (s₀, s₁) and the sign of s₁ provides a robust, geometry-respecting classifier.

The **Clifford bilinear form**:

```
⟨a, b⟩_Cl = a₀b₀ − a₁b₁
```

is used in memory retrieval scoring alongside cosine similarity (30% / 70% blend). This gives the retrieval metric a Minkowski geometry rather than the flat Euclidean one used by cosine similarity alone.

## 4. Spinor Projection and Chirality Balance

In practice, FANT 3 learns the mapping from hidden states to spinors end-to-end via a small linear projection:

```
proj_spinor: ℝ^dim → ℝ²      (a 2×dim matrix, bias=False)
```

This is initialized with `std=0.01` (very small weights) so that at training start the two spinor components have nearly equal magnitude and the chirality split is near 50/50. As training proceeds, the projection learns to encode semantically meaningful distinctions between instance-like (α) and schema-like (β) hidden states in the s₁ coordinate.

The **chirality balance** metric is:

```
chirality_balance = α_fill / (α_fill + β_fill) ∈ [0, 1]
```

A value of 0.5 indicates perfect balance. Values near 0 or 1 indicate starvation. Across all scales in the FANT 3 scale-ladder smoke test (2026-04-19), chirality balance ranged from 0.27 to 0.72, confirming that the spinor projection successfully separates the two populations at every model size.

## 5. The MASA Attention Parameterization

MASA (Multi-head Attention with Shared Atoms, arxiv:2508.04581) assembles per-layer attention matrices as a linear combination of shared atom matrices:

```
W_Q^l = Σᵢ  coef_q^l[i].sum() × A_Q[i]
```

where:
- `A_Q ∈ ℝ^(n_atoms × dim × dim)` are the shared atoms (one bank per model)
- `coef_q^l ∈ ℝ^(n_atoms × coef_rank)` are the per-layer coefficients

The parameter savings come from the fact that atom matrices are shared:
- Classic per-layer: `4 × n_layers × dim²`
- MASA: `4 × n_atoms × dim² + 4 × n_layers × n_atoms × coef_rank`

For `n_atoms=5`, `coef_rank=8`, `n_layers=20`, `dim=1024`, the savings are approximately:
- Classic: 4 × 20 × 1024² ≈ 83.9 M parameters in attention projections
- MASA: 4 × 5 × 1024² + 4 × 20 × 5 × 8 ≈ 21.0 M + 3.2 K ≈ 21.0 M parameters
- Savings: ~75%

## 6. ETF (Equiangular Tight Frame) Geometry

A simplex ETF (Equiangular Tight Frame) in ℝ^d with k vectors is a set {m₁, …, mₖ} satisfying:

```
‖mᵢ‖ = 1      for all i
mᵢ · mⱼ = −1/(k−1)   for all i ≠ j
```

This is the **maximally separated** configuration for k unit vectors in ℝ^d, achievable when k ≤ d+1. The "neural collapse" phenomenon (Papyan et al. 2020; arxiv:2412.00884) shows that final-layer classifier weights converge to exactly this geometry during training — the model discovers the ETF by gradient descent.

The FANT 3 router heads (`megapool_proj`, `level_proj`) are frozen to the ETF after `etf_freeze_after_step` training steps. The `simplex_etf(k, dim)` function computes this configuration via:

1. `M = I_k − (1/k) J_k` (centered identity: pairwise cosine = −1/(k−1))
2. SVD to extract the rank-(k−1) principal subspace
3. Pad with zeros to full `dim`, normalize rows
4. Apply a random orthogonal rotation (for non-axis-aligned initialization)

## 7. Partial RoPE (Rotary Position Embedding)

RoPE (Rotary Position Embedding, Su et al. 2021) encodes position by rotating Q and K in 2D subspaces of head dimension:

```
RoPE(x, pos)_d = x_d cos(pos × θ_d) − x_{d+1} sin(pos × θ_d)   for even d
```

where `θ_d = 10000^(−2d / head_dim)`. Full RoPE applies this to all head dimensions. Phi-4-Mini style **partial RoPE** applies it only to the first `rope_partial × head_dim` dimensions (`rope_partial=0.25` in FANT 3), leaving the remaining dimensions unrotated. This reduces the aliasing between position and content signals.

An important implementation detail: the `cos` and `sin` tensors must be explicitly cast to the input dtype (`x.dtype`, typically bf16 (bfloat16)) before the rotation. Without this cast, the default f32 output of `torch.cos`/`torch.sin` silently promotes `x` to f32, causing a dtype mismatch with the value tensor in PyTorch's fused scaled-dot-product attention kernel. This bug was diagnosed and fixed in `fant3/model/attention.py` on 2026-04-19.

## 8. Echo-State Network (Reservoir Computing) Theory

The reservoir computing module uses an echo-state network (ESN) (Jaeger 2001; Maass 2002). The key property is the **echo state property**: for a reservoir with recurrent weight matrix W_res and spectral radius ρ < 1, the network state h(t) is a deterministic function of the input history, regardless of the initial state. The convergence speed is governed by 1 − ρ.

At the **edge of chaos** (ρ ≈ 1, Bertschinger and Natschläger 2004), the reservoir maximizes its information-processing capacity while remaining stable. FANT 3 initializes the reservoir at ρ = 0.95, just inside the stable region.

The frozen reservoir is initialized once (seed=42) and never updated during training. Only the mossy-fiber projection (in → expand) and the Purkinje readout (expand → out) are learned. This gives the reservoir module a fixed computation budget (~12 M parameters) regardless of model scale, which is why the config specifies `cerebellum_in_dim=768` rather than `dim` as the input dimension.
